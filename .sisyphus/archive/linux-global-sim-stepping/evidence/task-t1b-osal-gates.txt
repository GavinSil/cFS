The stepping hooks are inserted at critical synchronization boundaries:
1. TaskDelay - allows stepping to control time progression
2. QueueReceive - allows stepping to manage message delivery timing
3. BinSemTake - allows stepping to manage synchronization points

When CFE_SIM_STEPPING is disabled, the #ifdef guards ensure:
- Hook calls do NOT compile into the binary
- No performance overhead in flight build
- No linker dependencies on hook implementations

When CFE_SIM_STEPPING is enabled, the hooks:
- Compile into the OSAL library
- Require implementation in stepping subsystem (T2)
- Provide deterministic control points for simulation
