Startup safety mechanism
1) System readiness gate (gate 2):
   - Field: system_ready_for_stepping (initialized false)
   - Set to true by: CFE_PSP_SimStepping_Core_MarkSystemReadyForStepping()
   - Called after: ES completes startup, apps loaded

2) TaskDelay takeover gate (gate 3):
   - Field: taskdelay_takeover_enabled (initialized false)
   - Set to true by: MarkSystemReadyForStepping()

3) Session active gate (gate 6):
   - Field: session_active (initialized false)
   - True only after first BeginStepSession() call

Three-phase behavior
Phase 1 (Startup, before ready):
  - system_ready = false
  - Result: All TaskDelay calls use wall-clock (no blocking)

Phase 2 (Ready, before first step):
  - system_ready = true, takeover_enabled = true
  - session_active = false
  - Result: TaskDelay still uses wall-clock (no session)

Phase 3 (Active stepping):
  - system_ready = true, takeover_enabled = true, session_active = true
  - Result: Eligible TaskDelay calls use PSP blocking wait

Startup log markers
- "CFE_ES_Main entering OPERATIONAL state" indicates startup completion
- Stepping only usable after this point

Interpretation vs T4 acceptance:
