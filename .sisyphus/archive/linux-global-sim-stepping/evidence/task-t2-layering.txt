     * OS_PosixStepping_Hook_BinSemTake() → calls shim_ReportEvent()
   
   - TIME: cfe/modules/time/fsw/src/cfe_time_stepping.c
     * CFE_TIME_Stepping_Hook_TaskCycle() → calls shim_ReportEvent()
     * CFE_TIME_Stepping_Hook_1HzBoundary() → calls shim_ReportEvent()
     * CFE_TIME_Stepping_Hook_ToneSignal() → calls shim_ReportEvent()
   
   - SCH: apps/sch/fsw/src/sch_stepping.c (via shim header)
     * Reports SCH_SEMAPHORE_WAIT, SCH_MINOR_FRAME, SCH_MAJOR_FRAME events

2) Shim layer (fact dispatch):
   - File: psp/fsw/modules/sim_stepping/cfe_psp_sim_stepping.c
   - Function: CFE_PSP_SimStepping_Shim_ReportEvent()
   - Role: Validates event, dispatches on event_kind, forwards to core Report* functions
   - No state machine logic, no socket handling

3) Core layer (state machine):
   - File: psp/fsw/modules/sim_stepping/cfe_psp_sim_stepping_core.c
   - Static instance: stepping_core (module-static)
   - Functions:
     * CFE_PSP_SimStepping_Core_BeginStepSession()
     * CFE_PSP_SimStepping_Core_IsStepComplete()
     * CFE_PSP_SimStepping_Core_ReportTaskDelay()
     * CFE_PSP_SimStepping_Core_ReportQueueReceive()
     * CFE_PSP_SimStepping_Core_ReportBinSemTake()
     * CFE_PSP_SimStepping_Core_ReportTimeTaskCycle()
     * CFE_PSP_SimStepping_Core_ReportSchMinorFrame()
     * (etc.)

4) Control channel adapters:
   - Inproc: CFE_PSP_SimStepping_InProc_* functions → call Core_* APIs
   - UDS: CFE_PSP_SimStepping_UDS_Service() → dispatches to InProc_* functions

Layering verification
Command: grep -R "SimStepHook_\|SimStepCore_\|trigger_set\|ack\|complete" psp/fsw/modules/sim_stepping/ 2>/dev/null
Result: Only hook and core files contain these symbols; OSAL/PSP/TIME/SCH only use shim ABI

No state machine duplication:
- OSAL hooks do not maintain trigger state
- TIME hooks do not maintain trigger state  
- SCH hooks do not maintain trigger state
- Only stepping_core maintains trigger_count, acks_expected, acks_received

Interpretation vs T2 acceptance:
- PASS: Low layer (OSAL/PSP/TIME/SCH) only calls hook/shim, no socket/trigger-set details
- PASS: State machine logic concentrated in native-only stepping core
- PASS: No state machine duplication in adapters

Conclusion
T2 hook/shim/core layering verification PASSED. Architecture follows thin hook + dispatcher + core state machine pattern.
