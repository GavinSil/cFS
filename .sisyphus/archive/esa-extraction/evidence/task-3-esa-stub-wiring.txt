## KEY INSIGHT

**Symbol Precedence via Link Order**: When multiple object files/libraries define
the same symbol, the linker uses the FIRST occurrence in the link command line.
By placing ut_esa_api_stubs.a before ut_psp_api_stubs.a, the ESA stub's 
CFE_PSP_SimStepping_Shim_ReportEvent symbol satisfies the reference, and the
PSP version is never linked (it's considered redundant/duplicate).

This is the canonical linker behavior and requires no wrapper functions or
complicated orchestration — just correct link order.
